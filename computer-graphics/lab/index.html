Lab 01
Drawing a Straight Line using C Program
1.1 Objective

To write a C program that draws a straight line on the screen using the basic parametric line equation, utilizing the putpixel() function available in BGI graphics.

1.2 Theory

A line segment between two points P1(x1, y1) and P2(x2, y2) can be described using the parametric form:

x(t) = x1 + t · (x2 - x1)
y(t) = y1 + t · (y2 - y1)
where t ∈ [0, 1], sampled at intervals of 1 / max(|dx|, |dy|)
1.3 Source Code
#include <stdio.h>
#include <graphics.h>
#include <math.h>

void drawLine(int x1, int y1, int x2, int y2) {
    int dx = x2 - x1, dy = y2 - y1;
    int steps = (abs(dx) > abs(dy)) ? abs(dx) : abs(dy);
    float xInc = (float)dx / steps;
    float yInc = (float)dy / steps;
    float x = x1, y = y1;
    for (int i = 0; i <= steps; i++) {
        putpixel((int)round(x), (int)round(y), WHITE);
        x += xInc; y += yInc;
    }
}

int main() {
    int gd = DETECT, gm;
    initgraph(&gd, &gm, "");
    drawLine(50, 50, 350, 200);
    getch(); closegraph();
    return 0;
}

Lab 02
Line Drawing using DDA Algorithm
2.1 Objective

To implement the Digital Differential Analyzer (DDA) algorithm in C for rasterizing a line segment between two given endpoints on a pixel grid.

2.2 Theory
Case |dx| >= |dy| (shallow): x increments by 1, y += m (slope)
Case |dy| > |dx| (steep): y increments by 1, x += 1/m

Generalized: steps = max(|dx|,|dy|), xInc = dx/steps, yInc = dy/steps
2.3 Source Code
#include <stdio.h>
#include <graphics.h>
#include <math.h>

void DDA(int x1, int y1, int x2, int y2) {
    int dx = x2 - x1, dy = y2 - y1;
    int steps = (abs(dx) > abs(dy)) ? abs(dx) : abs(dy);
    float xInc = (float)dx / steps;
    float yInc = (float)dy / steps;
    float x = x1, y = y1;
    for (int i = 0; i <= steps; i++) {
        putpixel((int)round(x), (int)round(y), CYAN);
        x += xInc; y += yInc;
    }
}

int main() {
    int gd = DETECT, gm;
    initgraph(&gd, &gm, "");
    DDA(40, 200, 360, 60);

Lab 03
Line Drawing using Bresenham's Algorithm
3.1 Objective

To implement Bresenham's Line Drawing Algorithm in C, using only integer arithmetic to determine pixel positions along a rasterized line.

3.2 Theory
Initial decision parameter: p0 = 2*dy - dx

If p < 0: plot (x+1, y), update p = p + 2*dy
If p >= 0: plot (x+1, y+1), update p = p + 2*dy - 2*dx

For general octants: use sign(dx) and sign(dy) to handle all directions.
3.3 Source Code
#include <stdio.h>
#include <graphics.h>
#include <stdlib.h>

void Bresenham(int x1, int y1, int x2, int y2) {
    int dx = abs(x2 - x1), dy = abs(y2 - y1);
    int sx = (x1 < x2) ? 1 : -1;
    int sy = (y1 < y2) ? 1 : -1;
    int err = dx - dy;
    while (1) {
        putpixel(x1, y1, YELLOW);
        if (x1 == x2 && y1 == y2) break;
        int e2 = 2 * err;
        if (e2 > -dy) { err -= dy; x1 += sx; }
        if (e2 <  dx) { err += dx; y1 += sy; }
    }
}

int main() {
    int gd = DETECT, gm;
    initgraph(&gd, &gm, "");
    Bresenham(30, 220, 380, 50);
    getch(); closegraph();
    return 0;
}

Lab 04
Geometric Translation (2D Transformation)
4.1 Objective

To implement geometric translation, a fundamental 2D affine transformation, which repositions every vertex of a shape by a fixed offset vector (Tx, Ty).

4.2 Theory
x' = x + Tx
y' = y + Ty

Homogeneous matrix form (3x3):
[ x' ] [ 1 0 Tx ] [ x ]
[ y' ] = [ 0 1 Ty ] [ y ]
[ 1 ] [ 0 0 1 ] [ 1 ]
4.3 Source Code
#include <stdio.h>
#include <graphics.h>

void drawPolygon(int x[], int y[], int n, int color) {
    setcolor(color);
    for (int i = 0; i < n - 1; i++)
        line(x[i], y[i], x[i+1], y[i+1]);
    line(x[n-1], y[n-1], x[0], y[0]);
}

void translate(int x[], int y[], int n, int tx, int ty) {
    for (int i = 0; i < n; i++) { x[i] += tx; y[i] += ty; }
}

int main() {
    int gd = DETECT, gm;
    initgraph(&gd, &gm, "");
    int x[] = {100,200,200,100};
    int y[] = {100,100,200,200};
    drawPolygon(x, y, 4, WHITE);
    translate(x, y, 4, 150, 80);
    drawPolygon(x, y, 4, CYAN);
    getch(); closegraph();
    return 0;
}

Lab 05
Midpoint Circle Drawing Algorithm
5.1 Objective

To implement the Midpoint Circle Algorithm (Bresenham's Circle) in C, which uses only integer arithmetic to rasterize a circle, exploiting 8-way symmetry to compute only 1/8th of the circle.

5.2 Theory

A circle with center (xc, yc) and radius r has the implicit equation f(x,y) = x^2 + y^2 - r^2 = 0.

Initial decision parameter: p0 = 1 - r

If p < 0: choose E (x+1, y), update p = p + 2x + 3
If p >= 0: choose SE (x+1, y-1), update p = p + 2(x-y) + 5

8-way symmetry: From one point (x,y), plot all 8 symmetric points:
(+/-x, +/-y), (+/-y, +/-x)

The algorithm stops when x > y (we've covered 45 degrees).

5.3 Source Code
#include <graphics.h>
#include <conio.h>

void plotCirclePoints(int xc, int yc, int x, int y) {
    putpixel(xc+x, yc+y, WHITE);
    putpixel(xc-x, yc+y, WHITE);
    putpixel(xc+x, yc-y, WHITE);
    putpixel(xc-x, yc-y, WHITE);
    putpixel(xc+y, yc+x, WHITE);
    putpixel(xc-y, yc+x, WHITE);
    putpixel(xc+y, yc-x, WHITE);
    putpixel(xc-y, yc-x, WHITE);
}

void midpointCircle(int xc, int yc, int r) {
    int x = 0, y = r;
    int p = 1 - r;
    plotCirclePoints(xc, yc, x, y);
    while (x < y) {
        x++;
        if (p < 0) { p += 2*x + 3; }
        else { y--; p += 2*(x-y) + 5; }
        plotCirclePoints(xc, yc, x, y);
    }
}

int main() {
    int gd = DETECT, gm;
    initgraph(&gd, &gm, "");
    midpointCircle(300, 200, 50);
    midpointCircle(300, 200, 100);
    midpointCircle(300, 200, 150);
    getch(); closegraph();
    return 0;
}

Lab 06
2D Rotation Transformation
6.1 Objective

To implement 2D rotation in C, which turns an object around a fixed pivot point by a specified angle theta. Positive angles rotate counterclockwise.

6.2 Theory
x' = x*cos(theta) - y*sin(theta)
y' = x*sin(theta) + y*cos(theta)

Matrix form:
[ x' ] [ cos(t) -sin(t) 0 ] [ x ]
[ y' ] = [ sin(t) cos(t) 0 ] * [ y ]
[ 1 ] [ 0 0 1 ] [ 1 ]

Rotation about arbitrary point (cx, cy):
1. Translate pivot to origin: T(-cx, -cy)
2. Rotate about origin: R(theta)
3. Translate back: T(cx, cy)
6.3 Source Code
#include <graphics.h>
#include <cmath>
#include <conio.h>
#define PI 3.14159265

void rotatePoint(int x, int y, int cx, int cy, float angle, int &xOut, int &yOut) {
    float rad = angle * PI / 180.0;
    float cosA = cos(rad), sinA = sin(rad);
    int xRel = x - cx, yRel = y - cy;
    xOut = cx + (int)(xRel * cosA - yRel * sinA + 0.5f);
    yOut = cy + (int)(xRel * sinA + yRel * cosA + 0.5f);
}

int main() {
    int gd = DETECT, gm;
    initwindow(640, 480, "2D Rotation");
    // Draw triangle, rotate about centroid by 60 degrees
    getch(); closegraph();
    return 0;
}

Lab 07
2D Scaling Transformation
7.1 Objective

To implement 2D scaling in C, which changes the size of an object by multiplying coordinates by scaling factors sx and sy.

7.2 Theory
x' = fx + (x - fx) * sx
y' = fy + (y - fy) * sy

Matrix form:
[ x' ] [ sx 0 0 ] [ x ]
[ y' ] = [ 0 sy 0 ] * [ y ]
[ 1 ] [ 0 0 1 ] [ 1 ]

Uniform scaling: sx = sy - shape preserved
Differential scaling: sx != sy - object stretches/distorts
7.3 Source Code
#include <graphics.h>
#include <cmath>
#include <conio.h>

void scalePoint(int x, int y, int fx, int fy, float sx, float sy, int &xOut, int &yOut) {
    xOut = fx + (int)((x - fx) * sx + 0.5f);
    yOut = fy + (int)((y - fy) * sy + 0.5f);
}

Lab 08
3D Rotation About an Arbitrary Axis
8.1 Objective

To implement 3D rotation about an arbitrary axis using Rodrigues' Rotation Formula in C, visualized on a 2D screen using oblique projection.

8.2 Theory
Rodrigues' Formula:
v' = v*cos(theta) + (u x v)*sin(theta) + u*(u . v)*(1 - cos(theta))

Where u = unit axis vector, v = point to rotate
u x v = cross product, u . v = dot product

Steps:
1. Translate so axis passes through origin
2. Normalize axis direction to unit vector u
3. Apply Rodrigues' formula
4. Translate back

Oblique projection (3D to 2D):
screen_x = world_x + world_z * 0.5 * cos(30 deg)
screen_y = world_y + world_z * 0.5 * sin(30 deg)
8.3 Source Code (excerpt)
#include <graphics.h>
#include <cmath>
#include <conio.h>
#define PI 3.14159265

struct Point3D { int x, y, z; };

void rotateAboutAxis(Point3D &p, Point3D a1, Point3D a2, float angle) {
    float px=p.x-a1.x, py=p.y-a1.y, pz=p.z-a1.z;
    float ax=a2.x-a1.x, ay=a2.y-a1.y, az=a2.z-a1.z;
    float len=sqrt(ax*ax+ay*ay+az*az);
    float ux=ax/len, uy=ay/len, uz=az/len;
    float rad=angle*PI/180.0, c=cos(rad), s=sin(rad);
    float dot=px*ux+py*uy+pz*uz;
    float cx=uy*pz-uz*py, cy=uz*px-ux*pz, cz=ux*py-uy*px;
    p.x=(int)(px*c+cx*s+ux*dot*(1-c)+a1.x+0.5f);
    p.y=(int)(py*c+cy*s+uy*dot*(1-c)+a1.y+0.5f);
    p.z=(int)(pz*c+cz*s+uz*dot*(1-c)+a1.z+0.5f);
}

Lab 09
Cohen-Sutherland Line Clipping
9.1 Objective

To implement the Cohen-Sutherland line clipping algorithm in C, which efficiently determines the visible portion of a line segment inside a rectangular clipping window.

9.2 Theory
Region codes (4 bits):
Bit 3 (8) = TOP (y < yMin)
Bit 2 (4) = BOTTOM (y > yMax)
Bit 1 (2) = RIGHT (x > xMax)
Bit 0 (1) = LEFT (x < xMin)
0000 = INSIDE

Trivial Accept: both codes = 0000
Trivial Reject: code1 AND code2 != 0
Otherwise: clip against one edge, replace endpoint, repeat
9.3 Source Code
#include <graphics.h>
#include <conio.h>

const int INSIDE=0, LEFT=1, RIGHT=2, BOTTOM=4, TOP=8;
int xMin=150, yMin=100, xMax=450, yMax=300;

int computeCode(int x, int y) {
    int code = INSIDE;
    if (x < xMin) code |= LEFT;
    if (x > xMax) code |= RIGHT;
    if (y < yMin) code |= TOP;
    if (y > yMax) code |= BOTTOM;
    return code;
}

void cohenSutherland(int x1, int y1, int x2, int y2) {
    int c1=computeCode(x1,y1), c2=computeCode(x2,y2), accept=0;
    while(1) {
        if(!c1 && !c2) { accept=1; break; }
        else if(c1 & c2) break;
        else {
            int cOut = (c1!=0)?c1:c2, x, y;
            if(cOut&TOP) { x=x1+(x2-x1)*(yMin-y1)/(y2-y1); y=yMin; }
            else if(cOut&BOTTOM) { x=x1+(x2-x1)*(yMax-y1)/(y2-y1); y=yMax; }
            else if(cOut&RIGHT) { y=y1+(y2-y1)*(xMax-x1)/(x2-x1); x=xMax; }
            else { y=y1+(y2-y1)*(xMin-x1)/(x2-x1); x=xMin; }
            if(cOut==c1) { x1=x; y1=y; c1=computeCode(x1,y1); }
            else { x2=x; y2=y; c2=computeCode(x2,y2); }
        }
    }
    if(accept) { setcolor(YELLOW); line(x1,y1,x2,y2); }
}

Lab 10
Sutherland-Hodgman Polygon Clipping
10.1 Objective

To implement the Sutherland-Hodgman polygon clipping algorithm, which clips an entire polygon against a rectangular window by processing one edge at a time.

10.2 Theory
Four cases for each edge (S to P):
1. S inside, P inside - Output P
2. S inside, P outside - Output intersection I
3. S outside, P outside - Output nothing
4. S outside, P inside - Output I, then P

Processing order: Left - Right - Bottom - Top
Each stage's output becomes the next stage's input.
10.3 Source Code (excerpt)
#include <graphics.h>
#include <vector>
using namespace std;

struct Point { int x, y; };
int xMin=150, yMin=100, xMax=450, yMax=300;

bool inside(Point p, int edge) {
    switch(edge) {
        case 0: return p.x >= xMin;
        case 1: return p.x <= xMax;
        case 2: return p.y >= yMin;
        case 3: return p.y <= yMax;
    }
    return false;
}

vector<Point> clipAgainstEdge(vector<Point> poly, int edge) {
    vector<Point> result;
    int n = poly.size();
    for(int i=0; i<n; i++) {
        Point cur=poly[i], nxt=poly[(i+1)%n];
        bool ci=inside(cur,edge), ni=inside(nxt,edge);
        if(ci && ni) result.push_back(nxt);
        else if(ci && !ni) result.push_back(intersect(cur,nxt,edge));
        else if(!ci && ni) {
            result.push_back(intersect(cur,nxt,edge));
            result.push_back(nxt);
        }
    }
    return result;
}

Lab 11
Bezier Curve
11.1 Objective

To implement a cubic Bezier curve in C, which generates a smooth curve from 4 control points using Bernstein basis polynomials.

11.2 Theory
B(t) = (1-t)^3*P0 + 3t(1-t)^2*P1 + 3t^2(1-t)*P2 + t^3*P3

Bernstein basis polynomials:
B0(t) = (1-t)^3 weight for P0
B1(t) = 3t(1-t)^2 weight for P1
B2(t) = 3t^2(1-t) weight for P2
B3(t) = t^3 weight for P3

Properties:
- Passes through P0 (t=0) and P3 (t=1)
- Tangent at start = direction P0 to P1
- Tangent at end = direction P2 to P3
- Curve lies inside the convex hull of control points
- Moving any control point affects the ENTIRE curve (global control)
11.3 Source Code
#include <graphics.h>
#include <cmath>
#include <conio.h>

void bezierCurve(int x[4], int y[4], int steps=100) {
    for(int i=0; i<=steps; i++) {
        float t=(float)i/steps, u=1-t;
        float b0=u*u*u, b1=3*t*u*u;
        float b2=3*t*t*u, b3=t*t*t;
        int px=(int)(b0*x[0]+b1*x[1]+b2*x[2]+b3*x[3]+0.5f);
        int py=(int)(b0*y[0]+b1*y[1]+b2*y[2]+b3*y[3]+0.5f);
        putpixel(px, py, YELLOW);
    }
}

Lab 12
B-Spline Curve
12.1 Objective

To implement a uniform cubic B-spline curve in C, which provides local control - each control point affects only a small region of the curve.

12.2 Theory
Uniform cubic B-spline segment:
Bi(t) = (1/6) * [
(1-t)^3*Pi + (3t^3 - 6t^2 + 4)*Pi+1 +
(-3t^3 + 3t^2 + 3t + 1)*Pi+2 + t^3*Pi+3
]

Key differences from Bezier:
- Local control: changing Pi affects at most 4 segments
- C2 continuous (smoother joins)
- Does NOT pass through control points
- n control points produce (n-3) segments
12.3 Source Code
#include <graphics.h>
#include <cmath>
#include <conio.h>
#define NUM_POINTS 6

void bSpline(int cx[], int cy[], int n, int steps=50) {
    setcolor(WHITE);
    for(int i=0; i<n-1; i++) line(cx[i],cy[i],cx[i+1],cy[i+1]);
    setcolor(YELLOW);
    for(int i=0; i<n-3; i++) {
        for(int j=0; j<=steps; j++) {
            float t=(float)j/steps, t2=t*t, t3=t2*t;
            float b0=(1.0f/6.0f)*(-t3+3*t2-3*t+1);
            float b1=(1.0f/6.0f)*(3*t3-6*t2+4);
            float b2=(1.0f/6.0f)*(-3*t3+3*t2+3*t+1);
            float b3=(1.0f/6.0f)*t3;
            int px=(int)(b0*cx[i]+b1*cx[i+1]+b2*cx[i+2]+b3*cx[i+3]+0.5f);
            int py=(int)(b0*cy[i]+b1*cy[i+1]+b2*cy[i+2]+b3*cy[i+3]+0.5f);
            putpixel(px, py, YELLOW);
        }
    }
}

Use
Practical Applications in Computer Graphics
Line Drawing & Circles
  • Vector graphics - SVG, PDF, PostScript rasterizers use Bresenham-based algorithms
  • CAD software - AutoCAD, SolidWorks use integer line drawing for wireframe previews
  • Game engines - Early 2D engines (Doom, Wolfenstein) used Bresenham for raycasting
  • Printer drivers - Laser/inkjet firmware implements Bresenham for nozzle control
Transformations
  • Animation - Translation, rotation, scaling form the basis of keyframe interpolation
  • GUI systems - Window managers apply transforms for dragging, scrolling, panning
  • Robotics & vision - Image registration uses affine transforms to align photos
  • GIS - Map panning uses translation and scaling of coordinate systems
Clipping & Curves
  • Viewport culling - Clipping prevents drawing objects outside the visible area
  • Font rendering - TrueType uses Bezier curves; OpenType adds B-splines
  • Animation paths - Bezier/B-splines define smooth camera trajectories
  • Industrial design - Car body and aircraft surfaces use B-spline/NURBS curves
